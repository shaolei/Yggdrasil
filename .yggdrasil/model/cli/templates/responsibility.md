# Templates Responsibility

Default templates and platform rules ‚Äî used by `yg init`.

**In scope:**

- **default-config.ts:** `DEFAULT_CONFIG` ‚Äî YAML string for default yg-config.yaml. Node types as object keyed by type name (module, service, library, infrastructure), each with a description string. Three artifacts: responsibility.md (always required, included_in_relations), interface.md (required when has_incoming_relations, included_in_relations), internals.md (never required). Quality thresholds for min_artifact_length, max_direct_relations, context_budget.
- **platform.ts:** `installRulesForPlatform(projectRoot, platform)` ‚Äî writes rules to platform-specific path; `PLATFORMS`, `Platform` type. Unknown platform falls through to generic.
- **rules.ts:** `AGENT_RULES_CONTENT` ‚Äî canonical agent rules (operating manual); hand-tuned, do not generate programmatically. Used by platform.ts. Split internally into three cognitive sections (`CORE_PROTOCOL`, `OPERATIONS`, `KNOWLEDGE_BASE`) optimized for LLM attention patterns, concatenated into a single export. Core Protocol covers quick start protocol with semantic search pre-step (step 0 ‚Äî intent-based navigation when semantic search tools are available) and two graph-first paths (understand via `build-context`, assess impact via `yg impact`), five core rules (Rule 2: "The graph is the specification; code implements it" ‚Äî knowledge absorption, self-sufficiency, per-file timing, planning integration; plus "capture why NOT" with rejected alternatives), failure states, escape hatch, environment check, and subagent delegation (requires subagents to read .yggdrasil/agent-rules.md as first instruction + explicit deliverables: code, graph nodes, validate passing). Operations covers conversation lifecycle (using `yg preflight`), modify code/graph checklists, greenfield graph-first workflow (6-step: aspects ‚Üí flows ‚Üí nodes ‚Üí build-context ‚Üí implement), reverse engineering with aspect identification guidance (3-instance heuristic, domain/architectural/concurrency taxonomy, step 6b for code anchors in yg-node.yaml), bootstrap mode, drift resolution (with drift triage prioritization), graph audit (consistency + completeness checks including aspect exceptions verification), and error recovery. Includes "BEFORE ENDING ANY RESPONSE" self-audit: did I modify code? If yes ‚Üí did I update graph before moving to the next file? Knowledge Base covers graph structure (with node type guidance including `infrastructure` type for guards/middleware/interceptors), artifact structure (3-level: responsibility/interface/internals), context assembly, information routing (with decisions.md rejected-alternatives template), creating aspects/flows (with embedded `exceptions` guidance for node-specific deviations within aspect entries), operational rules with enhanced completeness test, CLI reference, and quick routing table.
- **graph-schemas/:** yg-node.yaml, yg-aspect.yaml, yg-flow.yaml ‚Äî schemas for each graph layer, copied to .yggdrasil/schemas/ during init

**Out of scope:**

- Init logic (cli/commands/init)
- Templates in .yggdrasil/schemas/ (generated by init, not part of source)
